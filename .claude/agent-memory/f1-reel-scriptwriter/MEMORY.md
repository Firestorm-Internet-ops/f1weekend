## User Preferences (Inferred)
- User wants honest, precise critique — no softening.
- Reviews should be exhaustive: hook ratings, loop analysis, CTA ratings, rewrites, bug flags.
- F1 telemetry metaphors should be natural, not forced.

## Script Writing Principles (Confirmed Effective)
- hero-design works because: fragmented hook, consistent philosophy thread, payoff line as closer.
- hero-build works because: human cost embedded in opener, serialised narrative, hard metrics as receipt.
- Templates fail when: Scene 1 is identical across all instances, variables don't adapt to content.
- CTAs must engineer forward action, not request a follow.
- Curiosity loops must stay open at Scene 4 — never close fully.
